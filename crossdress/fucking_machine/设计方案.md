| 项目 | 说明 |
|:----|------|
| 控制板 | ESP32-S 开发板 × 1（上次剩的 520板） |

## 三、机构设计

### 3.1 曲柄连杆原理

```
        ┌───────┐
        │ 电机   │  ← 减速电机
        │ 输出轴  │
        └───┬───┘
            │
        ┌───┴───┐
        │ 偏心轮  │  ← 曲柄（圆盘或偏心杆），偏心距 = 行程一半
        │   /    │
        │  ●     │
        └──┬────┘
           │ ← 连杆（连接销）
           │
         ┌─┴─┐
         │ 滑块 │  ← 直线导轨上滑动
         │ 🍆 │  ← 假阳具接口
         └───┘

   ┌─── 行程 S = 2 × 偏心距 e
   │
   例：偏心距 5cm → 行程 10cm
```

### 3.2 速度与行程关系

| 电机转速 | 偏心距 | 行程 | 频率 | 效果 |
|:-------:|:-----:|:---:|:----:|:----:|
| 100rpm | 4cm | 8cm | 1.7次/秒 | 慢速深插 |
| 100rpm | 6cm | 12cm | 1.7次/秒 | 慢速超深 |
| 200rpm | 4cm | 8cm | 3.3次/秒 | 中速标准 |
| 200rpm | 6cm | 12cm | 3.3次/秒 | 中速深插 |

**建议电机规格：0-200rpm 可调**（通过 PWM 调速，可不是降到零速）

> 实际 PWM 启动电压需要一个最低启动值（电机启动时最低要一定的PWM占空比，一般为 20-30%）。低于此值电机不转。

### 3.3 3D 打印建议结构

```
┌──────────────────────────────────────────┐
│  底座                                     │
│  ┌────────────────────────────────┐      │
│  │ 电机固定座 ┌────────┐          │      │
│  │  ┌────┐   │ 滑块   │          │      │
│  │  │电机│───→◄════►  │          │      │
│  │  │    │   │ 直线   │ ←限位开关 │      │
│  │  └────┘   │ 导轨   │          │      │
│  │           └────────┘          │      │
│  └────────────────────────────────┘      │
│                                          │
│  ┌────────────────────────────────┐      │
│  │ 电控仓（ESP32 + 驱动板）       │      │
│  └────────────────────────────────┘      │
└──────────────────────────────────────────┘
```

## 四、电路图

### 4.1 选定方案：IRF520（需外挂续流二极管 ⚠ 必须）

```
                    ┌──────────────────┐
  12V ───── + ──────┤ IRF520 模块       │
                    │                  │
                    │ V+ (螺丝端子)      │
                    │ V- (螺丝端子) ── GND│
                    │                  │
  ESP32 GPIO18 ────┤ SIG (信号输入)    │
                    │                  │
                    │ OUT (螺丝端子) ─┬── 电机(-) │
                    │ V+ ─────────────┤── 电机(+) │
                    └──────────────────┘
                          │
                          ├─── SS34 阴极 ←─── 电机(+)
                          └─── SS34 阳极 ←─── 电机(-)
                          （二极管并联在电机两端，阴极接正极）


  ┌────────────────────────────────────────────┐
  │  ESP32 接线                                 │
  │                                            │
  │  电源：                                     │
  │    ESP32 5V/VIN ── 5V 电源（USB供电即可）    │
  │    ESP32 GND ──── IRF520 V- ── 12V电源GND  │
  │    12V+ ──────── IRF520 V+                 │
  │                                            │
  │  信号：                                     │
  │    GPIO18 ──── IRF520 SIG (PWM 调速)       │
  │    GPIO27 ──── 限位开关1 (前限位，INPUT_PULLUP)│
  │    GPIO26 ──── 限位开关2 (后限位，INPUT_PULLUP)│
  │                                            │
  │  电机：                                     │
  │    电机(+) ──── IRF520 V+                  │
  │    电机(-) ──── IRF520 OUT                  │
  │    电机两端并联 SS34 (阴极→电机+, 阳极→电机-)│
  │    电机两端并联 100μF + 0.1μF 滤波电容      │
  └────────────────────────────────────────────┘

  ⚠ IRF520 驱动电机注意事项：
  - MOSFET 只能单向驱动 → 曲柄机构只需正转，没问题
  - 启动瞬间电流大 → PWM 不要从 0 跳 100，做 2 秒软启动
  - IRF520 没有散热片持续 2A+ 会烫 → 长时间高负载注意散热
  - 电机停止时务必设 PWM=0（不能浮空）
  - 连续运行 30 分钟建议休息一下免得 IRF520 过热
```

## 五、固件架构（MicroPython / ESP32）

### 主循环逻辑

```
开机 → 创建 WiFi AP (ESP32_Fucker_xxxx)
     → 启动 WebSocket 服务器
     → 提供 Web 控制页面
     → 循环：
        接收 WebSocket 控制命令
        更新 PWM 占空比
        检查限位开关
        发送当前状态回客户端
```

### 控制协议（JSON over WebSocket）

```json
// 客户端→ESP32
{
  "cmd": "set_speed",     // 设定速度
  "value": 70             // 0-100%
}
{
  "cmd": "set_mode",      // 设定模式
  "mode": "sine"          // constant / sine / random / pulse
}
{
  "cmd": "set_time",      // 定时
  "minutes": 15
}
{
  "cmd": "stop"           // 急停
}
{
  "cmd": "start"          // 启动/恢复
}

// ESP32→客户端（每秒）
{
  "speed": 70,
  "mode": "sine",
  "running": true,
  "remaining": 840,      // 剩余秒数
  "limit_hit": false,    // 限位触发
  "uptime": 360,         // 已运行秒数
  "temp": 32.5           // 电机温度（如接DS18B20）
}
```

### 运行模式

| 模式 | 描述 | PWM 曲线 |
|:----|------|:--------:|
| **constant** | 恒定速度 | ▁▁▁▁▁▁▁ |
| **sine** | 正弦波（快慢交替） | ▁▃▆▇▆▃▁▃▆ |
| **random** | 随机变速 | 随机阶梯变化 |
| **pulse** | 脉冲式（快→停→快） | █▁▁█▁▁█ |
| **ramp** | 渐加速 | ▁▂▃▄▅▆▇█ |
| **wave** | 波浪（长周期快慢） | ▃▆▇▆▃▁▃▆▇▆▃▁ |

## 七、连接控制方式

### 电源方案（已定）

**独立 12V 电源适配器**，跟灌肠器完全分离。

| 设备 | 供电 | 备注 |
|:----|:----|:----|
| 电机 | 12V 2A 电源适配器 → IRF520 V+ | **+ 独立供电，不共用** |
| ESP32 | USB 供电（手机充电头/充电宝） | 避免电机纹波干扰 |
| GND | ESP32 GND ← → IRF520 GND ← → 12V GND | **共地，信号回路必需** |

> 买一个 **12V 2A 电源适配器（DC 5.5×2.1）** 淘宝 ¥8-10
> 把插头剪了直接接线到 IRF520 V+ 和 GND

### 模式A：ESP32 自建热点（AP 模式）—— 推荐

```
ESP32 ─── 创建 WiFi：SSID: "Fucker_XXXX" 密码: "12345678"
              │
        手机连接此 WiFi
              │
        浏览器访问 192.168.4.1
```

- 不需要路由器，随便在哪儿都能用
- 控制距离 = WiFi 信号范围（室内 ~10-15m）

### 模式B：连接家里路由器（STA 模式）

```
手机 ── 家里WiFi ── 路由器 ── ESP32
```

- 可以用局域网内任意设备控制
- 如果路由器有公网IP/端口映射，甚至可以远程控制

**建议：固件同时支持两种模式，启动时优先连已知 WiFi，连不上则自建热点。**

## 八、安全保护

| # | 保护 | 实现 | 动作 |
|:-:|------|------|------|
| 1 | **限位开关** | 微动开关 × 2，行程两端 | 到限位后停 PWM，反向复位 |
| 2 | **Web 急停按钮** | 大红按钮，全屏可见 | 立即停 PWM，锁死 |
| 3 | **看门狗定时器** | ESP32 硬件 WDT | 死循环自动复位 |
| 4 | **超时停机** | 设定时间到 → 自动停 | 蜂鸣器提示 |
| 5 | **速度渐变** | 启动/停止 2 秒渐变 | 防止突然插入/拔出 |
| 6 | **电源掉电保护** | 断电即停，MOSFET 截止 | 安全 |

## 九、扩展方向

### 联动灌肠器（终极形态）

两台 ESP32 通过 **MQTT 或简单 HTTP** 通信：

```
灌肠器 ──↔── 炮机
   │           │
   ├ 充气完成→ 开始运动
   ├ 水压骤升→ 降低速度
   ├ 脱落→     紧急停止
   └ 排空→     逐渐减速
```

### 更多花样

- 节奏模式：上传自定义速度曲线
- 声音响应：麦克风检测 → 速度跟随（刺激叫越猛越快）
- 远程控制：通过 MQTT 让第三方遥控
- 震动反馈：加上震动马达，和炮机交替运转
