## Relay Service Resolution - 2026-05-01 → 2026-05-05

### Problem
OpenClaw Browser Relay WebSocket connection failed with "did not receive a valid HTTP response" and HTTP requests returning 426/500 errors. Chrome plugin unable to connect to `http://127.0.0.1:18792/`.

### Root Causes Identified
1. **websockets library v11+ API change**: `process_request` callback signature changed from `process_request(path, headers)` to `process_request(connection, request)` where `request` is a `Request` object, not a headers dict.
2. **HTTP vs WebSocket on same port**: The `websockets` library's `process_request` is only called for WebSocket upgrade requests, not plain HTTP GET requests. Plain HTTP requests were being rejected with 426 by the library itself.
3. **Mixed Python environments**: MinGW Python (`python3`) lacked `websockets` module; Windows Python (`python`) had it.
4. **Multiple conflicting relay processes**: PIDs 1084, 15844, 19160, 29460 left in various states.

### Solutions Attempted
- Tried `final-relay.py` with `process_request` callback - failed due to HTTP rejection
- Tried `final-relay-debug.py` - websockets v11+ compatibility issues
- Tried `relay-service.py` - same library limitations

### Final Working Solution (2026-05-01 20:53+)
**File**: `relay-manual.py`  
**Lines**: 8222  
**Approach**: Manual implementation using `asyncio.start_server()` without external websockets library

### Key Implementation Details
1. **Manual HTTP request parsing** - `HTTPRequest` class parses method, path, headers
2. **Manual WebSocket handshake** - SHA1(Sec-WebSocket-Key + GUID) base64 encoded
3. **Manual WebSocket frame encoding** - FIN bit, opcode, mask, payload length handling
4. **Manual WebSocket frame decoding** - Masking key application, opcode dispatch
5. **Protocol detection** - Checks `Connection: Upgrade` + `Upgrade: websocket` + `Sec-WebSocket-Key` header

### Lessons Learned
1. **Don't trust library defaults** - websockets v11 changed behavior significantly
2. **Read the source/docs carefully** - `process_request` semantics differ by version
3. **Manual protocol implementation** is sometimes simpler than fighting library abstractions
4. **Environment consistency** matters - multiple Pythons cause confusion
5. **Clean up processes** - leftover PIDs block ports and confuse debugging
